Repo idea: andrewinua/validator-transparency-dashboard • Open on GitHub Pages

Methodology. “Epoch performance (proxy)” is not true uptime. The dashboard takes the last 6 entries of epochCredits from getVoteAccounts, looks at how much voting credits increased from epoch to epoch, and compares each epoch to your best recent one. Those relative values are averaged into a 0–100% score over the last 5 epochs. It’s a quick way to see whether a validator has been voting consistently recently, but real uptime still requires node-level monitoring and logs.
