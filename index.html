Source model. Public blocks use live Solana RPC + Stakewiz + Trillium + Jito proxy. The stability block uses backend-collected snapshot history combined with current public inputs.
Methodology. Recent voting consistency is derived from vote-credit changes in recent vote account data. Each recent epoch is normalized against the strongest epoch in the same recent window, converted into a relative percentage, and the displayed headline value is the average of the latest 5 such observations. Higher means steadier recent voting, not guaranteed reliability.
History model. Opening a validator page can register it for backend tracking. Snapshot history is then collected server-side on a schedule and reused for all viewers of that validator page. It is not browser-local and does not depend on who is currently visiting the dashboard.
