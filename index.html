Method separation. This dashboard now distinguishes between: public live validator data, public APY/pool data, and a separate local tracking layer for personal browser-based stability monitoring.

Methodology. “Voting consistency” is a relative consistency score derived from epochCredits returned by getVoteAccounts. The dashboard takes the last six entries, looks at how voting credits changed from epoch to epoch, and compares each epoch with the best recent one. Those relative values are averaged into a 0–100% score over the last five epochs.

Next step worth adding later. Real public commission-change history and longer operational history should be powered by a dedicated backend/public-history endpoint rather than by browser-local tracking.
