Methodology. “Voting consistency” is a relative consistency score derived from epochCredits returned by getVoteAccounts. The dashboard takes the last six entries, looks at how voting credits changed from epoch to epoch, and compares each epoch with the best recent one. Those relative values are averaged into a 0–100% score over the last five epochs. It’s a quick way to see how consistently a validator has been voting in recent epochs, while detailed uptime still comes from node-level monitoring and logs.
